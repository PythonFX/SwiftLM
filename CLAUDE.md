## Build Commands

```bash
# Full build (submodules + cmake + Metal kernels + Swift)
./build.sh

# Swift-only build (after initial build.sh)
swift build -c release

# Run unit tests
swift test

# Run the server
.build/release/SwiftLM --model mlx-community/Qwen2.5-3B-Instruct-4bit --port 5413

# Build SwiftBuddy (macOS)
./build_swiftbuddy.sh

./run_benchmark.sh
```

### Prerequisites
- macOS 14+, Apple Silicon (M1–M5)
- Xcode Command Line Tools
- `cmake` via Homebrew
- Metal Toolchain: `xcodebuild -downloadComponent MetalToolchain`

## Architecture

SwiftLM is a native Swift inference server that runs MLX models on Apple Silicon via Metal, exposed through an OpenAI-compatible HTTP API.

### Target Layout (Package.swift)

| Target | Type | Description |
|--------|------|-------------|
| `SwiftLM` | Executable | CLI HTTP server — all API routes, model lifecycle, inference orchestration |
| `MLXInferenceCore` | Library | Shared inference engine (used by both CLI and iOS app) — model download/storage, architecture probing, ALM |
| `DFlash` | Library | Block-diffusion speculative decoding engine with custom Metal kernels |
| `SwiftBuddy` | Executable | iOS/macOS SwiftUI companion app |
| `DFlashKernelBench` | Executable | DFlash kernel micro-benchmark tool |
| `SwiftLMTestSTFT` | Executable | STFT audio profiling test |

### Critical Files
- `Sources/SwiftLM/Server.swift` — Monolithic server (146KB). All HTTP routes, inference pipeline, streaming, KV cache lifecycle.
- `Sources/MLXInferenceCore/InferenceEngine.swift` — Core inference loop shared between CLI and iOS app.
- `Sources/DFlash/DFlashRuntime.swift` + `DFlashKernels.swift` — Speculative decoding execution and Metal kernels.

### Submodules (SharpAI Forks)
- `mlx-swift/` — Forked from Apple's MLX Swift with custom C++ primitives for SSD streaming and TurboQuant Metal kernels. **Not interchangeable with upstream.**
- `mlx-swift-lm/` — Forked from Apple's MLX LLM library with GPU/CPU layer partitioning, MoE flush gate, MTP head support.

### Data Flow
```
HTTP Request (OpenAI API)
  → Server.swift (route matching, payload parsing)
    → MLXInferenceCore (model loading, tokenization, inference engine)
      → mlx-swift-lm (model architecture, attention, KV cache)
        → mlx-swift (MLX primitives, Metal GPU kernels)
          → Metal GPU (Apple Silicon)
```

## Key Patterns

- **Server.swift is monolithic by design** — it handles argument parsing, all API routes, model lifecycle, and inference orchestration. When modifying API behavior, work within this file.
- **MLXInferenceCore is shared** between macOS CLI and iOS app — changes affect both platforms. `StrictConcurrency` is enabled.
- **Model architecture support** lives in the `mlx-swift-lm` submodule, not in this repo. New model support requires changes there.
- **Integration tests are shell/Python scripts** in `tests/` — they exercise the running server via HTTP. `tests/test-server.sh` is the most comprehensive.
- **Profiling results** go to `docs/profiling/` and are auto-generated by `scripts/profiling/profile_runner.py`.
- **DFlashKernelsOptimized.swift** is excluded from the build — do not reference it.
- **SwiftBuddy's `.xcodeproj`** is git-ignored. Run `python3 SwiftBuddy/generate_xcodeproj.py` to regenerate locally.

## Testing

```bash
# Unit tests (XCTest)
swift test

# Integration tests (require running server or auto-launch one)
bash tests/test-server.sh          # Full API endpoint coverage
bash tests/test-vision.sh          # VLM image inputs
bash tests/test-speculative.sh     # Dual-model speculative decoding
bash tests/test-dflash.sh          # DFlash block-diffusion
bash tests/test-audio.sh           # Audio-language models
bash tests/test-omni.sh            # Omni (audio+text) models

# Benchmarks
python3 tests/run_4models_benchmark.py   # 4-model TPS/RAM benchmark
python3 -u scripts/profiling/profile_runner.py --model <model> --contexts "512,40000,100000"
```
